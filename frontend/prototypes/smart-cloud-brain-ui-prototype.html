前端界面总体规划

覆盖患者端、医生端、管理端，优先满足完整诊疗闭环与 P0 验收点。

完整诊疗闭环

从患者输入主诉开始，到医生确认病历、处方审核，再回到患者查看记录。

统一通过 gateway-service /api 与 /ws
1. 注册登录

患者、医生、管理员三类身份登录，JWT 与角色路由守卫。

2. 智能分诊

患者输入主诉，AI 推荐科室与医生，失败时允许手动选择。

3. 在线挂号

选择医生、日期、号源，创建挂号并可查看或取消。

4. 问诊病历

医生录入对话，AI 流式生成结构化病历，医生确认保存。

5. 处方审核

医生开药，AI 审核风险，HIGH 风险触发实时通知。

6. 记录回看

患者查看挂号、电子病历、处方和审核摘要。
