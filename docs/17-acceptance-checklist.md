# 阶段验收清单

## 主业务链

- 患者注册并登录。
- 患者输入轻症描述，获得 AI 分诊建议。
- 患者根据推荐结果完成预约挂号。
- 医生登录后查看待接诊挂号。
- 医生生成 AI 病历草稿并确认保存。
- 医生开具处方并调用 AI 审核。
- 患者查看病历和处方记录。

## 必交材料

- 源代码：后端、Vue 3 Web 端、uni-app 安卓端、SQL、Postman。
- 文档：分工清单、系统设计、数据库设计、接口设计、测试计划、部署说明、AI 使用说明。
- 截图/录屏：三端页面、Postman 接口、数据库表、完整业务链演示。
- 测试：`mvn test`、`npm test` 或 `pnpm test`，以及 WebRunner 性能报告。

## 明确不做

- 不做住院。
- 不做真实支付、医保、药房库存和真实发药。
- 不把 AI 输出作为真实医疗诊断结论。
