# 五人小组分工清单

项目按“端 + 能力专题”组织协作，实际成员姓名可在提交前替换。

| 角色 | 主要职责 | 交付物 |
|---|---|---|
| 组长 / 后端负责人 | 维护总体进度、接口设计、Gateway 和主业务闭环 | 后端接口、Postman、答辩主线 |
| AI 服务负责人 | 维护 `ai-service`、Mock Provider、Prompt 和真实模型替换点 | AI 服务代码、AI 使用说明、降级策略 |
| Web 前端负责人 | 维护患者端、医生端、管理端 | Web 页面、截图、联调记录 |
| 移动端负责人 | 维护 uni-app 患者移动端 | `patient-mobile`、移动端演示截图 |
| 测试与部署负责人 | 维护 Kingbase SQL、测试、Docker、文档和答辩材料 | 测试记录、部署说明、演示脚本 |

## 协作要求

- 每个成员在独立分支开发，避免覆盖他人模块。
- 公共 API 兼容优先，接口变更需要同步文档和 Postman。
- 前端不能写死业务假数据，必须通过 Gateway 调后端。
- AI 可以使用 Mock Provider，但必须走 `ai-service`。
- 阶段验收至少保留截图、录屏或命令输出记录。
