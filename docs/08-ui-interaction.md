# 页面与交互文档

## 1. 页面列表

| 页面 | 用户 | 核心功能 |
|---|---|---|
| 登录页 | 患者、医生、管理员 | 登录并保存 Token |
| 患者首页 | 患者 | 快速进入分诊、挂号、记录 |
| 智能分诊 | 患者 | 输入主诉并展示后端 AI 建议 |
| 在线挂号 | 患者 | 查看数据库号源并预约 |
| 我的挂号 | 患者 | 查看和取消挂号 |
| 我的病历 | 患者 | 查看医生确认后的病历 |
| 我的处方 | 患者 | 查看处方和审核结果 |
| 医生工作台 | 医生 | 查看本人待接诊队列 |
| 病历编辑 | 医生 | 生成草稿、编辑、保存 |
| 处方审核 | 医生 | 开方、审核、确认处方 |
| 通知列表 | 医生 | 查看 WebSocket 风险通知 |
| 管理首页 | 管理员 | 查看管理入口 |
| 基础数据 | 管理员 | 维护科室、医生、药品、字典 |
| Prompt/知识库 | 管理员 | 维护 AI 相关配置 |
| AI 排班 | 管理员 | 生成建议并发布号源 |
| 分诊工作台 | 管理员 | 查看、改派、关闭分诊记录 |

## 2. 交互原则

- 登录后根据角色进入对应端。
- 页面刷新后从本地 Token 恢复登录态，再拉取后端数据。
- 表格和详情页数据均通过接口加载。
- 创建、取消、保存、发布等操作成功后重新拉取后端数据。
- AI 生成过程展示 Loading 或生成状态。
- AI 输出必须标注为辅助建议，并保留人工确认步骤。

## 3. 三端关键流程

### 患者端

```text
登录 -> 智能分诊 -> 查看推荐 -> 选择号源 -> 预约挂号 -> 查看病历/处方
```

### 医生端

```text
登录 -> 待接诊队列 -> 生成病历 -> 保存病历 -> 处方审核 -> 保存处方 -> 接收通知
```

### 管理端

```text
登录 -> 维护基础数据 -> 生成排班建议 -> 发布号源 -> 分诊工作台改派/关闭
```

## 4. 数据来源要求

- 号源列表来自 `appointment_slot`。
- 医生队列来自 `registration`。
- 病历来自 `medical_record`。
- 处方来自 `prescription` 和 `prescription_item`。
- 通知来自 `notification_message` 和 WebSocket。
- AI 展示内容来自后端 `ai-service` 返回结果。
