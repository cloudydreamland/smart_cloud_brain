# 测试文档

## 1. 功能测试用例

| 用例 | 步骤 | 预期 |
|---|---|---|
| 患者注册 | 输入合法姓名、手机号、密码提交 | 注册成功 |
| 重复注册 | 使用相同手机号注册 | 返回手机号已存在 |
| 患者登录 | 输入正确手机号密码 | 返回 JWT 和用户信息 |
| 医生登录 | 输入医生账号密码 | 进入医生工作台 |
| 医生列表 | 选择心内科筛选 | 返回心内科医生 |
| 智能分诊 | 输入“胸痛伴气短” | 推荐心内科和相关医生 |
| 创建挂号 | 选择医生和时间段 | 挂号成功 |
| 取消挂号 | 取消未就诊挂号 | 状态变为 CANCELLED |
| AI 生成病历 | 输入医患对话 | 自动回填病历字段 |
| 保存病历 | 修改并提交病历表单 | 病历保存成功 |
| AI 处方审核 | 输入药品列表 | 返回风险等级和建议 |
| 保存处方 | 审核后提交处方 | 处方保存成功 |
| 患者查看记录 | 进入病历和处方页 | 能查看本人记录 |

## 2. 接口测试用例

| 接口 | 测试点 | 预期 |
|---|---|---|
| `/api/patient/register` | 参数完整 | 成功 |
| `/api/patient/register` | 手机号为空 | 400 |
| `/api/patient/login` | 密码错误 | 400 或业务错误 |
| `/api/patient/info` | 不带 Token | 401 |
| `/api/doctor/list` | 按科室筛选 | 返回指定科室医生 |
| `/api/triage/consult` | 主诉为空 | 400 |
| `/api/registration/create` | 重复挂号 | 409 |
| `/api/medical-record/generate` | AI 超时 | 600 或降级响应 |
| `/api/prescription/check` | 药品为空 | 400 |
| `/api/prescription/list` | 患者查询 | 仅返回本人处方 |

## 3. 边界测试

- 手机号不足 11 位。
- 密码长度不足 6 位。
- 主诉文本超过 1000 字。
- 预约过去时间。
- 同一患者同一医生同一时间重复挂号。
- 处方药品明细超过 20 条。
- AI 返回空内容。
- AI 返回非 JSON 内容。
- JWT 过期。
- 数据库中医生被禁用后仍尝试挂号。

## 4. 异常测试

| 场景 | 预期 |
|---|---|
| 数据库连接失败 | 后端启动失败并输出明确日志 |
| AI 服务不可用 | 返回降级提示，不影响手动业务 |
| 后端接口 500 | 前端展示错误提示，不白屏 |
| 网络超时 | 前端提示请求超时，可重试 |
| Token 被篡改 | 返回 401 |

## 6. 性能测试建议

| 指标 | 建议 |
|---|---|
| 普通列表接口 | 小于 500ms |
| 普通写接口 | 小于 1s |
| AI 分诊接口 | 小于 10s |
| AI 病历生成接口 | 小于 15s |
| AI 处方审核接口 | 小于 10s |
| 分页大小 | 默认 10 或 20 |
| 并发测试 | 20 到 50 个并发用户可作为课程项目演示基线 |
