# Smart Cloud Brain 前端静态原型

本目录是第一阶段静态 HTML 原型，用于审阅患者端、医生端、管理端的信息架构和视觉方向。当前阶段不依赖 Vite、不依赖后端接口，也不修改 `frontend/apps` 或 `frontend/packages` 下的真实业务代码。

## 文件

- `patient-web.html`：患者端核心工作区原型。
- `doctor-web.html`：医生端诊疗工作台原型。
- `admin-web.html`：管理端运营维护工作台原型。
- `shared-style.css`：三端共享设计系统与响应式样式。
- `README.md`：原型说明。

## 预览方式

直接在浏览器中打开任意 HTML 文件即可预览，例如：

```text
C:\Users\Riddle\Desktop\smart_cloud_brain\docs\frontend-prototypes\patient-web.html
```

三个页面顶部都有轻量原型导航，可在患者端、医生端、管理端之间切换。

## 设计系统方向

- 风格：医疗 SaaS / 诊疗工作台，强调清晰、稳重、效率和适中信息密度。
- 色彩：医疗青绿作为主色，搭配中性底色、暖色提示、成功/警告/危险/信息状态色，避免单调蓝紫或灰蓝。
- 组件：统一按钮、表单、表格、标签、通知、空状态、错误状态、分段控件和响应式栅格。
- 布局：侧边导航 + 顶部上下文 + 主工作区，不做营销 hero，不使用渐变球或装饰光斑，不做卡片套卡片。
- 响应式：桌面端优先工作台效率，移动端改为纵向堆叠并保留表格横向滚动。

## 三端重点

- 患者端：突出“AI 分诊 -> 推荐号源 -> 预约确认 -> 我的挂号 -> 病历处方”的连续流程，并显式设计暂无号源、暂无病历和请求失败状态。
- 医生端：突出“待接诊队列 -> 当前患者上下文 -> AI 病历草稿 -> 医生确认保存 -> 处方录入与 AI 审核 -> 风险通知”的诊疗闭环。
- 管理端：突出"科室医生维护 -> AI 排班号源发布 -> 分诊工作台 -> 字典/药品/Prompt/知识库维护 -> 批量编辑"的运营维护效率。
