# 产品需求文档 PRD

## 1. 产品定位

智慧云脑诊疗平台是面向课程实训的智慧诊疗综合平台。产品重点不是替代真实医院系统，而是通过完整诊疗业务闭环展示 Spring Boot 3、Spring Cloud、Vue 3、KingbaseES、RabbitMQ、WebSocket 和 AI 服务化的综合实践。

## 2. 用户与场景

| 用户 | 典型场景 |
|---|---|
| 患者 | 登录、输入症状、查看 AI 分诊建议、预约挂号、查看病历和处方 |
| 医生 | 查看待接诊患者、生成病历、保存病历、审核处方、接收风险通知 |
| 管理员 | 维护基础数据、生成排班建议、发布号源、处理分诊记录 |

## 3. P0 功能

| 模块 | 功能 |
|---|---|
| 登录认证 | 患者、医生、管理员登录，JWT 鉴权 |
| 患者端 | AI 分诊、号源预约、挂号取消、病历处方查看 |
| 医生端 | 待接诊队列、AI 病历生成、病历保存、处方审核、通知查看 |
| 管理端 | 基础数据维护、AI 排班、分诊工作台 |
| AI 服务 | 智能分诊、病历草稿、处方审核、排班建议 |
| 通知 | RabbitMQ 事件和 WebSocket 推送 |
| 部署 | Docker Compose 一键启动 |

## 4. 页面范围

- 患者端：首页、智能分诊、号源列表、我的挂号、我的病历、我的处方。
- 医生端：登录、待接诊队列、病历编辑、处方审核、通知列表。
- 管理端：基础数据、Prompt、知识库、字典、AI 排班、分诊工作台。
- 移动端：患者基础演示入口。

## 5. 数据与真实性

- 患者、医生、管理员、科室、药品、Prompt、知识库、字典、排班和号源均来自数据库。
- 挂号、病历、处方、通知由演示流程实时写入。
- 前端不写死业务演示数据。
- AI 当前使用 Mock Provider，但所有 AI 结果都通过后端链路产生。

## 6. 验收指标

- 三端可登录并完成主流程。
- 后端指定 Maven 离线命令编译通过。
- 前端三端可构建。
- Docker Compose 可启动核心服务。
- 数据库可查询到初始化数据和流程产生的数据。
- 答辩能明确说明真实流程与 Mock AI 的边界。
