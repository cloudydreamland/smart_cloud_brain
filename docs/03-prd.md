# 产品需求文档 PRD

## 1. 产品定位

东软智慧云脑诊疗平台是面向课程实践的智慧诊疗综合平台。产品重点不是替代真实医院系统，而是通过完整诊疗业务闭环展示 Spring Boot 3 + Vue 3 前后端分离开发方式，以及 AI 能力如何嵌入传统医疗业务流程。

## 4. 交互流程

### 4.1 患者分诊挂号

1. 患者登录。
2. 进入智能分诊页。
3. 输入主诉文本。
4. 点击“智能分诊”。
5. 页面展示 Loading。
6. 系统返回推荐科室、推荐医生和推荐原因。
7. 患者点击医生卡片上的“去挂号”。
8. 选择时间段并提交。
9. 挂号成功后跳转“我的挂号”。

### 4.2 医生问诊病历

1. 医生登录。
2. 进入医生工作台。
3. 选择待诊患者。
4. 进入问诊病历页。
5. 录入模拟医患对话。
6. 点击“AI 生成病历”。
7. AI 生成结果自动回填结构化病历表单。
8. 医生修改并保存。

### 4.3 医生处方审核

1. 医生进入处方开具页。
2. 添加药品明细。
3. 点击“AI 辅助审核”。
4. 系统展示风险等级、相互作用和建议。
5. 医生根据建议修改处方或确认保存。
6. 处方和审核记录入库。

## 5. 输入输出规则

| 字段 | 规则 |
|---|---|
| 手机号 | 必填，11 位数字，唯一 |
| 密码 | 必填，至少 6 位，后端加密存储 |
| 姓名 | 必填，2 到 50 字 |
| 主诉 | 必填，建议 10 到 1000 字 |
| 挂号时间 | 必须为未来时间 |
| 药品名称 | 必填 |
| 药品剂量 | 必填，例如 `100mg` |
| 用药频次 | 必填，例如 `每日一次` |
| 病历主诉 | 必填 |
| 初步诊断 | 必填 |

## 6. 权限规则

| 资源 | 患者 | 医生 |
|---|---|---|
| 患者个人信息 | 仅本人可见 | 接诊相关患者可见 |
| 挂号记录 | 仅本人可见 | 与本人相关挂号可见 |
| 病历记录 | 仅本人可见 | 创建医生或接诊医生可见 |
| 处方记录 | 仅本人可见 | 创建医生可见 |
| AI 分诊记录 | 仅本人可见 | 原则上医生不可直接查看，除非关联挂号 |
| AI 审核记录 | 患者可查看结果摘要 | 医生可查看完整审核信息 |

## 7. 优先级划分

### P0 必须实现

- 项目初始化。
- 患者注册登录。
- 医生登录。
- 医生列表和详情。
- 在线挂号。
- 智能分诊。
- JWT 鉴权。
- 医生工作台。
- AI 病历生成。
- 病历保存和查询。
- 处方开具。
- AI 处方审核。
- 患者查看病历和处方。
- WebSocket 实时告警。
- SSE 或 WebSocket 流式病历生成。
- Pinia 前端状态机优化。
- AI Prompt 模板管理。
- 双端分离部署。
- AI 微服务拆分，使用 Spring Cloud Feign 完成服务间调用。

### P2 可选增强

- 管理端维护基础数据。
