# 开发计划文档

## 1. 开发阶段划分

| 阶段 | 任务 | 优先级 | 产出 |
|---|---|---|---|
| 第 1 阶段 | 项目初始化 | P0 | 前后端脚手架、数据库连接、Swagger、CORS |
| 第 2 阶段 | 基础框架能力 | P0 | 统一响应、全局异常、JWT、Axios 封装 |
| 第 3 阶段 | 认证与基础数据 | P0 | 患者注册登录、医生登录、科室和医生数据 |
| 第 4 阶段 | 挂号业务 | P0 | 医生列表、挂号创建、挂号列表、取消挂号 |
| 第 5 阶段 | 智能分诊 | P0 | AI 分诊接口、分诊页面、推荐医生跳转挂号 |
| 第 6 阶段 | 医生工作台和病历 | P1 | 待诊患者、AI 病历生成、病历保存 |
| 第 7 阶段 | 处方与审核 | P1 | 处方开具、AI 审核、处方保存 |
| 第 8 阶段 | 综合联调 | P1 | 完整演示流程、Postman 记录、截图/录屏 |
| 第 9 阶段 | 拓展优化 | P2 | WebSocket、SSE、Prompt 模板、部署优化 |

## 2. 预计开发顺序

1. 创建 Spring Boot 3 项目。
2. 创建 Vue 3 + TypeScript 项目。
3. 配置 MySQL、JPA、Knife4j、CORS。
4. 实现 `Result<T>`、错误码、全局异常。
5. 实现 JWT 登录认证。
6. 实现患者注册登录。
7. 初始化科室和医生数据。
8. 实现医生列表和详情。
9. 实现挂号创建、列表、取消。
10. 实现智能分诊后端 Service 和前端页面。
11. 实现医生工作台。
12. 实现 AI 病历生成和病历保存。
13. 实现处方开具和 AI 审核。
14. 完成端到端联调。
15. 编写测试记录和项目汇报材料。

## 3. MVP 版本范围

MVP 必须支持一条完整演示链路：

```text
患者注册登录 -> 智能分诊 -> 在线挂号 -> 医生查看患者 -> AI 生成病历 -> AI 处方审核 -> 保存处方 -> 患者查看记录
```

MVP 必选能力：

- 患者注册、登录、个人信息。
- 医生登录。
- 科室和医生列表。
- 智能分诊。
- 在线挂号。
- 医生工作台。
- AI 病历生成和保存。
- AI 处方审核和保存。
- JWT 鉴权。
- Swagger/Knife4j 文档。

## 4. 后续迭代方向

| 方向 | 说明 |
|---|---|
| 管理端 | 维护科室、医生、药品、系统字典 |
| WebSocket | 高风险处方实时通知医生 |
| SSE 流式生成 | 病历生成逐字展示 |
| 知识图谱 | 药品相互作用、疾病科室映射 |
| Prompt 模板 | 针对不同科室优化 AI 生成效果 |
| 微服务拆分 | 将 AI 能力层拆为独立服务 |
| Docker 部署 | 使用 Docker Compose 启动前端、后端、数据库 |
