# 09 开发计划

## 1. 当前目标

优先交付 P0 可演示闭环：患者完成 AI 分诊与预约挂号，医生完成病历与处方，管理员完成基础数据、字典、分诊工作台和 AI 排班发布。

## 2. 开发顺序

1. 补齐数据库结构和演示种子数据。
2. 完成认证、患者、医生、挂号基础链路。
3. 接入 `ai-service` Mock Provider，打通分诊、病历、处方审核。
4. 完成 RabbitMQ + WebSocket 风险通知。
5. 完成管理端基础数据、排班和分诊工作台。
6. 接入三端前端页面，确保全部通过 Gateway 调后端。
7. 完成 Docker Compose 启动和 README 复现说明。
8. 执行后端 Maven 编译、前端测试和三端构建。
9. 整理验收截图、数据库查询截图和答辩脚本。

## 3. 数据策略

- KingbaseES/PostgreSQL 兼容数据库是演示唯一数据源。
- `sql/kingbase_schema.sql` 提供基础数据和号源。
- 挂号、病历、处方、通知通过演示流程实时生成。
- AI 使用 Mock Provider，但结果必须由后端返回。

## 4. 非 P0 范围

- 不接入真实支付、医保、短信、药房库存、生产审计。
- 不接入真实医院生产数据。
- 不以 AI 医学准确率作为验收核心。
