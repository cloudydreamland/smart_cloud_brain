# 东软智慧云脑诊疗平台项目文档

本文档集基于项目任务描述整理，适合作为后续开发、联调、测试、答辩和迭代的基础材料。

文档口径说明：核心业务任务、成果要求、答辩要求和非功能要求以 `东软智慧云脑诊疗平台实践任务描述(1).docx` 为基础；图片中的 WebSocket、流式响应、Pinia 状态机、Prompt、双端分离部署和纯微服务架构六项任务全部作为必做交付项。

统一口径声明：本项目采用“基础诊疗闭环 + FY26 企业项目实训方案 + 补充必做任务”的统一口径。第一阶段以后端 `diagnosis-service + ai-service` 承载可运行主链路，其他服务目录保留为后续领域拆分和长期维护规划；前端通过 `/api` 代理访问业务能力。患者端、医生端、管理端均为必做入口。WebSocket 实时通知、AI 流式响应、Prompt 工程、双端分离部署、纯微服务演进六项任务全部作为 P0 必做交付，不再作为可选扩展项处理。管理端第一版聚焦科室、医生、药品、Prompt 模板和轻症知识库维护。

## 已确认范围

1. AI 服务提供方未明确：可使用本地 Mock、学校提供接口或第三方大模型 API。
2. 医生账号来源未明确：MVP 默认医生账号可由管理员端或初始化 SQL 维护。
3. 管理端为第三个必做使用入口，第一版覆盖科室、医生、药品、Prompt 模板和轻症知识库。
4. 是否涉及真实支付、医保、检查检验系统未明确：MVP 不包含支付、医保、LIS/PACS 对接。
5. 药品知识库来源未明确：MVP 使用本地药品基础数据加 AI 审核结果模拟，后续可接知识图谱。
6. 医疗合规边界未明确：MVP 定位为教学实践系统，AI 结果仅作辅助建议，最终由医生确认。

## 基础假设

- 项目为课程实践和演示系统，不用于真实医疗诊疗决策。
- 系统包含患者端、医生端、管理端三个必做使用入口。
- 后端第一阶段以 `diagnosis-service + ai-service` 跑通业务闭环，后续按认证、患者、医生、挂号、分诊、病历、处方、AI、通知等业务域逐步拆分。
- 管理端必做范围为基础数据维护、Prompt 模板和轻症知识库维护。
- 数据库目标使用 KingbaseES，开发演示可临时使用 MySQL 8 兼容脚本。
- 前后端通过 RESTful API 通信，接口统一返回 `Result<T>`。
- 登录认证使用 JWT。
- AI 接口尽量返回结构化 JSON；若 AI 服务不可用，系统回退到人工录入流程。
