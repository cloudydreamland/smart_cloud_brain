# 项目概述文档

## 1. 项目背景

随着人工智能、大数据和医疗信息化的发展，诊疗服务正在从传统的“医生经验驱动、人工操作为主”逐步走向“数据驱动、AI 辅助、全流程协同”。传统医疗信息化课程或实践项目通常只覆盖挂号、病历录入、药品管理等单一环节，难以体现诊前、诊中、诊后之间的业务闭环。

本项目围绕“AI + 诊疗”构建完整智慧诊疗实践平台，采用 Spring Boot 3 + Spring Cloud + Vue 3 前后端分离纯微服务架构，将患者管理、在线挂号、医生问诊、处方管理、病历归档与 AI 智能分诊、AI 处方审核、AI 病历生成能力结合起来，使学生能够完成从前端页面、网关路由、业务域微服务、数据库设计到 AI 能力集成的完整全栈训练。

统一口径声明：本项目采用“基础诊疗闭环 + FY26 企业项目实训方案 + 补充必做任务”的统一口径。后端从项目初始化起即采用 Spring Boot 3 + Spring Cloud 纯微服务架构，前端统一通过 `gateway-service` 访问业务能力；患者端、医生端、管理端均为必做入口。WebSocket 实时通知、AI 流式响应、Pinia 状态机、Prompt 工程、双端分离部署、纯微服务架构六项任务全部作为 P0 必做交付，不再作为可选扩展项处理。管理端除基础数据维护外，还需要覆盖 AI 医生排班和 AI 分诊台的最小可演示能力。

## 2. 项目目标

1. 构建一个可运行、可演示的智慧诊疗前后端分离纯微服务系统。
2. 实现患者注册登录、智能分诊、在线挂号、医生问诊、处方开具、病历生成等完整业务流程。
3. 将 AI 能力嵌入传统诊疗业务节点，而不是作为孤立演示功能存在。
4. 将患者端、医生端、管理端作为三端必做入口，其中管理端覆盖基础数据维护、AI 医生排班和 AI 分诊台。
5. 将 WebSocket、AI 流式响应、Pinia 状态机、Prompt 工程、双端分离部署、纯微服务架构六项任务作为 P0 必做交付。
6. 帮助学习者掌握 Vue 3、TypeScript、Spring Boot 3、Spring Cloud、RESTful API、JWT、服务治理、数据库设计和 AI 服务集成。
7. 支撑课程答辩、项目汇报、演示视频和联调测试记录提交。

## 3. 核心价值

| 价值类型 | 说明 |
|---|---|
| 教学价值 | 覆盖前端、后端、数据库、AI 接口、联调、测试、部署全流程 |
| 工程价值 | 体现企业级前后端分离和纯微服务项目组织方式 |
| 业务价值 | 模拟患者诊前、医生诊中、患者诊后查看记录的完整闭环 |
| AI 融合价值 | 展示 AI 分诊、AI 审核、AI 生成如何嵌入真实业务流程 |
| 架构价值 | 覆盖网关、注册发现、OpenFeign、熔断降级、服务数据边界和独立部署 |

## 5. 使用场景

1. 患者输入症状，系统调用 AI 推荐科室和医生。
2. 管理员在 AI 分诊台查看分诊记录、推荐理由和处理状态，必要时人工改派科室或医生。
3. 管理员通过 AI 医生排班生成排班建议，人工确认后发布号源。
4. 患者根据推荐结果和可预约号源选择医生和时间段完成挂号。
5. 医生登录后查看待接诊患者。
6. 医生录入医患对话，调用 AI 自动生成结构化病历草稿。
7. 医生编辑并确认病历，保存电子病历。
8. 医生开具处方，调用 AI 审核用药风险。
9. 患者查看自己的挂号记录、电子病历和处方记录。

## 6. 典型演示案例

管理员先维护科室、医生、药品、系统字典和 Prompt 模板，并通过 AI 医生排班生成心内科号源建议，人工确认后发布。患者注册登录后进入智能分诊页面，输入“胸痛伴气短，活动后加重”。系统调用 AI 分诊服务，推荐“心内科”和相关医生；管理员可在 AI 分诊台查看该分诊记录，必要时人工改派。患者选择医生完成在线挂号。医生登录医生端后，在工作台查看该患者，进入问诊页面，录入模拟医患对话，点击“AI 生成病历”，系统自动回填主诉、现病史、既往史、初步诊断、治疗建议等字段。医生确认后保存病历，并进入处方页面开具药品。提交前调用“AI 辅助审核”，系统返回用药建议、风险等级和相互作用提示。医生确认后保存处方，患者端可查看病历和处方记录。
