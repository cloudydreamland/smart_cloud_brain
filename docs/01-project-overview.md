# 项目概述文档

## 1. 项目背景

智慧云脑诊疗平台面向课程实训和项目答辩，模拟诊前分诊、在线挂号、医生接诊、病历生成、处方审核、风险通知和后台管理的完整闭环。项目重点是展示前后端分离、Spring Cloud 微服务、数据库建模、异步事件、实时通信和 AI 服务化能力。

本项目不是真实医院生产系统，AI 输出不作为真实诊断依据。

## 2. 项目目标

1. 构建可运行、可演示、可部署的智慧诊疗微服务系统。
2. 支持患者端、医生端、管理端三端核心流程。
3. 通过 Gateway 统一承接前端请求。
4. 使用 KingbaseES/PostgreSQL 兼容数据库作为唯一事实库。
5. 使用 RabbitMQ 和 WebSocket 完成处方风险通知。
6. 使用 `ai-service` 承载智能分诊、病历生成、处方审核和排班建议。
7. 当前 AI 使用 Mock Provider，保证演示稳定并保留真实模型替换点。

## 3. 核心价值

| 价值类型 | 说明 |
|---|---|
| 教学价值 | 覆盖前端、后端、数据库、微服务、消息队列、AI 接口和部署全流程 |
| 工程价值 | 展示 Gateway、OpenFeign、Nacos、RabbitMQ、Docker Compose 等企业级实践 |
| 业务价值 | 模拟患者、医生、管理员之间的真实诊疗协作 |
| 答辩价值 | 能清楚区分真实数据库流程和 Mock AI 输出 |

## 4. 用户角色

- 患者：登录、分诊、预约挂号、查看病历和处方。
- 医生：查看待接诊队列、生成病历、开具处方、接收风险通知。
- 管理员：维护基础数据、生成排班建议、发布号源、处理分诊工作台。

## 5. 核心演示链路

```text
患者登录
  -> 提交主诉并获得 AI 分诊建议
  -> 从数据库号源完成预约挂号
  -> 医生查看本人待接诊队列
  -> 医生生成并确认病历
  -> 医生执行处方审核并保存处方
  -> RabbitMQ/WebSocket 推送风险通知
  -> 患者查看病历和处方
  -> 管理员维护数据并发布排班号源
```

## 6. 交付物

- 后端 Spring Boot 多模块微服务。
- 前端 Vue 3 患者端、医生端、管理端。
- uni-app 患者移动端。
- KingbaseES 初始化 SQL。
- Docker Compose 部署配置。
- Postman 调试集合。
- 项目设计、部署、测试、验收和答辩文档。
