# 项目概述文档

## 1. 项目背景

随着人工智能、大数据和医疗信息化的发展，诊疗服务正在从传统的“医生经验驱动、人工操作为主”逐步走向“数据驱动、AI 辅助、全流程协同”。传统医疗信息化课程或实践项目通常只覆盖挂号、病历录入、药品管理等单一环节，难以体现诊前、诊中、诊后之间的业务闭环。

本项目围绕“AI + 诊疗”构建完整智慧诊疗实践平台，采用 Spring Boot 3 + Vue 3 前后端分离架构，将患者管理、在线挂号、医生问诊、处方管理、病历归档与 AI 智能分诊、AI 处方审核、AI 病历生成能力结合起来，使学生能够完成从后端 API、前端页面、数据库设计到 AI 能力集成的完整全栈训练。

## 2. 项目目标

1. 构建一个可运行、可演示的智慧诊疗前后端分离系统。
2. 实现患者注册登录、智能分诊、在线挂号、医生问诊、处方开具、病历生成等完整业务流程。
3. 将 AI 能力嵌入传统诊疗业务节点，而不是作为孤立演示功能存在。
4. 帮助学习者掌握 Vue 3、TypeScript、Spring Boot 3、RESTful API、JWT、数据库设计和 AI 服务集成。
5. 支撑课程答辩、项目汇报、演示视频和联调测试记录提交。

## 3. 核心价值

| 价值类型 | 说明 |
|---|---|
| 教学价值 | 覆盖前端、后端、数据库、AI 接口、联调、测试、部署全流程 |
| 工程价值 | 体现企业级前后端分离项目组织方式和分层架构 |
| 业务价值 | 模拟患者诊前、医生诊中、患者诊后查看记录的完整闭环 |
| AI 融合价值 | 展示 AI 分诊、AI 审核、AI 生成如何嵌入真实业务流程 |
| 扩展价值 | 可扩展知识图谱、流式生成、实时通知、管理端、微服务拆分 |

## 4. 目标用户

| 用户类型 | 说明 |
|---|---|
| 高校师生 | 用于课程设计、实验教学、综合实训和答辩展示 |
| 全栈开发学习者 | 学习 Spring Boot 3 + Vue 3 前后端分离开发 |
| AI 应用学习者 | 学习大模型 API 与传统业务系统的融合方式 |
| 医疗信息化学习者 | 理解挂号、处方、病历等基础医疗业务建模 |

## 5. 使用场景

1. 患者输入症状，系统调用 AI 推荐科室和医生。
2. 患者根据推荐结果选择医生和时间段完成挂号。
3. 医生登录后查看待接诊患者。
4. 医生录入医患对话，调用 AI 自动生成结构化病历草稿。
5. 医生编辑并确认病历，保存电子病历。
6. 医生开具处方，调用 AI 审核用药风险。
7. 患者查看自己的挂号记录、电子病历和处方记录。

## 6. 典型演示案例

患者注册登录后进入智能分诊页面，输入“胸痛伴气短，活动后加重”。系统调用 AI 分诊服务，推荐“心内科”和相关医生。患者选择医生完成在线挂号。医生登录医生端后，在工作台查看该患者，进入问诊页面，录入模拟医患对话，点击“AI 生成病历”，系统自动回填主诉、现病史、既往史、初步诊断、治疗建议等字段。医生确认后保存病历，并进入处方页面开具药品。提交前调用“AI 辅助审核”，系统返回用药建议、风险等级和相互作用提示。医生确认后保存处方，患者端可查看病历和处方记录。
