# 数据库设计文档

## 1. 数据库选型

验收环境以 KingbaseES 为准，开发环境可使用 PostgreSQL 兼容数据库。字段命名采用小写下划线，主键使用 `BIGINT GENERATED BY DEFAULT AS IDENTITY`。

主脚本：

```text
sql/kingbase_schema.sql
```

## 2. 设计原则

- KingbaseES 是唯一事实库。
- 前端不维护业务假数据。
- 各服务拥有清晰数据归属。
- 跨服务只保存逻辑 ID，不直接跨库 join。
- 业务流程数据由真实操作产生。
- 初始化脚本提供基础演示数据和号源。

## 3. 核心表

| 表 | 归属 | 说明 |
|---|---|---|
| `patient` | 患者 | 患者账号和基础资料 |
| `doctor` | 医生 | 医生账号、科室、职称和专长 |
| `department` | 医生/管理 | 科室主数据 |
| `admin_user` | 管理 | 管理员账号 |
| `doctor_schedule` | 医生 | 已发布医生排班 |
| `appointment_slot` | 医生/挂号 | 可预约号源 |
| `registration` | 挂号 | 患者预约记录 |
| `triage_record` | 分诊 | AI 分诊记录和人工分配结果 |
| `medical_record` | 病历 | 医生确认后的正式病历 |
| `prescription` | 处方 | 医生确认后的处方 |
| `prescription_item` | 处方 | 处方药品明细 |
| `prescription_check_record` | 处方 | AI 审核记录 |
| `notification_message` | 通知 | 医生端通知历史 |
| `outbox_event` | 处方/通知 | 待投递异步事件 |
| `prompt_template` | AI | Prompt 模板 |
| `ai_generation_log` | AI | AI 调用日志 |
| `drug` | 管理 | 药品和相互作用规则 |
| `knowledge_entry` | 管理/AI | 轻症知识库 |
| `system_dict` | 管理 | 系统字典 |

## 4. 初始化数据

初始化脚本应提供：

- 默认患者：`13800000001 / 123456`
- 默认医生：`doctor1 / 123456` 或 `13900000001 / 123456`
- 默认管理员：`admin / 123456`
- 科室、医生、药品、Prompt、知识库、字典。
- 已发布排班和 `remaining_capacity > 0` 的号源。

## 5. 流程数据

演示过程中产生：

- `triage_record`：患者提交分诊后生成。
- `registration`：患者预约后生成。
- `medical_record`：医生确认病历后生成。
- `prescription` 和 `prescription_item`：医生确认处方后生成。
- `prescription_check_record`：AI 处方审核后生成。
- `outbox_event` 和 `notification_message`：风险通知链路生成。

## 6. 验收查询建议

```sql
select * from appointment_slot where remaining_capacity > 0;
select * from registration order by id desc;
select * from triage_record order by id desc;
select * from medical_record order by id desc;
select * from prescription order by id desc;
select * from notification_message order by id desc;
```
